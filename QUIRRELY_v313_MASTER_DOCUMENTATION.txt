1. WHAT QUIRRELY IS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Quirrely is a writing voice analysis platform for English-language writers in
Commonwealth markets (Canada, UK, Australia, New Zealand). It identifies a
writer's natural voice profile using the LNCP (Literary-Narrative Classification
Protocol) engine and gives them tools to understand, develop, and stretch that
voice.

The product has three surfaces:
  - Web app (quirrely.com) — primary consumer interface
  - Chrome Extension v2.0.0 — passive analysis of writing across the web
  - Blog (91 posts) — SEO-driven acquisition funnel

The business model is freemium SaaS with CAD Stripe billing. The core
monetisation mechanism is STRETCH: a writing exercise system that locks
premium growth features behind the Pro and Authority tiers.

Geo-restriction: US users are redirected. Target markets: CA, UK, AU, NZ.

4. THE LNCP ENGINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

LNCP (Literary-Narrative Classification Protocol) is Quirrely's core IP.
It classifies written text into voice profiles and stances using a multi-phase
pipeline. The client-side version (Quinquaginta v3.8) runs fully offline in the
browser and Chrome extension.

VERSION: 3.8.0 (engine) / Quinquaginta v3.8 (classifier)

-- The 10 StyleAxis Profiles --

  ASSERTIVE     Direct, confident, declarative. Strong claims, low hedging.
  HEDGED        Measured, qualified. Acknowledges uncertainty throughout.
  CONVERSATIONAL  Informal, accessible. Reads as spoken, not written.
  FORMAL        Structured, authoritative. Follows conventions of formal writing.
  DENSE         Information-rich, compressed. High concept load per sentence.
  MINIMAL       Sparse, economical. Every word justified. Implication over statement.
  POETIC        Lyrical, image-driven. Sound and rhythm carry meaning.
  ANALYTICAL    Systematic, evidence-grounded. Cause-and-effect structure.
  INTERROGATIVE Question-driven. Opens rather than closes. Genuine uncertainty.
  LONGFORM      Expansive, elaborative. Suited to extended argument and digression.

  Note: Early architecture documents used "PARALLEL" for the 10th profile.
  ANALYTICAL is the canonical name as defined in lncp/engine/profiles.py.
  See backend/PROFILES_MANIFEST.md.

-- The 4 CertitudeAxis Stances --

  OPEN          Explores possibilities. Invites dialogue. Leaves room for doubt.
  CLOSED        Definitive conclusions. Makes strong, resolved statements.
  BALANCED      Weighs multiple perspectives. Acknowledges complexity fairly.
  CONTRADICTORY Holds opposing truths simultaneously. Embraces paradox.

-- Pipeline Phases --

  Phase 1   Compute baseline text metrics
  Phase 2   Sentence-level pattern extraction
  Phase 3   Profile scoring (all 10 profiles scored)
  Phase 4a  Certitude axis analysis
  Phase 4b  Stance refinement
  Phase 6   Final profile + delta synthesis

-- Key Files --

  lncp/engine/profiles.py    StyleAxis and CertitudeAxis enums, token maps
  lncp/engine/scoring.py     compute_basic_metrics, score_assertiveness,
                             score_density, score_formality, score_hedging
  lncp/engine/tokens.py      Token management and profile token sets
  lncp/engine/value.py       Value functions for profile scoring

-- HALO Safety System --

  Detects harmful, unsafe, or flagged content before analysis proceeds.
  Files: halo_bridge.py, halo_detector.py, halo_core_bridge.py,
         halo_feedback.py, halo_observer.py
  Schema: database/schema_halo.sql (also staged at backend/schema_halo.sql)

5. THE STRETCH SYSTEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STRETCH is the core monetisation mechanism. After a user completes 5 rounds
of analysis, they are offered STRETCH exercises: prompts designed to push their
writing into a different profile. Completing a STRETCH extends their trial by
+3 days and presents an upgrade CTA.

VERSION: 1.0.0

-- Exercise Structure --

  1 exercise = 5 cycles
  1 cycle    = 3 prompts
  Total:       15 prompted writing sessions per exercise

-- Prompt Dataset --

  File:    backend/stretch_prompts_base.json
  Count:   450 prompts
  Matrix:  10 profiles × 5 cycles × 3 prompt positions × 3 variants
  Fields:  target_profile, cycle_number, prompt_position, variant,
           story_starter, instruction, word_guidance, difficulty (1-5), tags

-- Keystroke Validation (Zero Tolerance) --

  Paste detection runs on both client and server.
  Client: assets/js/stretch-components.js (KeystrokeTracker class)
  Server: backend/stretch_api.py (KeystrokeValidator class)
  
  Rejection triggers: paste event, Ctrl+V / Cmd+V, drag-and-drop,
  burst typing (characters per second exceeds threshold),
  content length vs keystroke count mismatch.

-- API Endpoints (all at /api/stretch/) --

  GET  /eligibility/{user_id}                  Check if user can do STRETCH
  GET  /recommend/{user_id}                    Get recommended stretches
  POST /start                                  Begin a new exercise
  GET  /current/{user_id}                      Get active exercise
  GET  /exercise/{exercise_id}                 Get exercise details
  GET  /cycle/{exercise_id}/{cycle_number}     Get cycle details
  GET  /prompt/{exercise_id}/{cycle}/{prompt}  Get a specific prompt
  POST /input/{exercise_id}/{cycle}/{prompt}   Submit response (validates keystrokes)
  POST /complete-cycle/{exercise_id}/{cycle}   Mark cycle complete
  GET  /progress/{user_id}                     Get progress metrics
  GET  /history/{user_id}                      Get exercise history
  POST /abandon/{exercise_id}                  Abandon exercise
  GET  /cta/{user_id}                          Get upgrade CTA state
  POST /cta/clicked/{user_id}                  Record CTA click
  POST /cta/dismissed/{user_id}                Record CTA dismissal

-- Tier Access --

  free:      STRETCH disabled
  trial:     STRETCH enabled (drives the +3 day extension → upgrade)
  pro:       3 stretch types, opposite profiles only
  authority: 6 stretch types, adjacent + opposite profiles
  curator:   10 stretch types, all profiles

-- Prompt Resolver --

  Country groups: commonwealth (CA, UK, AU, NZ, IE), us
  Stance modifiers: OPEN, CLOSED, BALANCED, CONTRADICTORY
  Spelling substitutions: harbor↔harbour, color↔colour, traveled↔travelled

  Classes: StretchComponent, KeystrokeTracker, StretchAPIClient
  Used by: extension popup, standalone stretch pages

           calls and Chrome extension. Handled by auth_middleware.py.

  auth_middleware.py: JWT encode/decode, Bearer token validation

9. META ORCHESTRATOR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Version: 5.1.0
File:    lncp/meta/meta_orchestrator.py

The meta orchestrator is an autonomous optimisation layer. It receives signals
from 8 observers, evaluates system health, generates proposals for parameter
changes, and routes conversion signals to the appropriate CTA surfaces.

-- The 8 Observers --

  stretch_observer      STRETCH completion → conversion_ready signal.
                        Routes upgrade CTAs to users who just completed STRETCH.

  revenue_observer      MRR tracking, ARR projections, subscription health.
                        Monitors tier transitions and revenue concentration.

  retention_observer    Churn intent detection. Monitors cancel clicks, low
                        usage, payment failures. Triggers retention interventions
                        (pause offer, 50% discount, win-back email).

  funnel_observer       Full acquisition funnel (landing → analysis → signup →
                        trial → STRETCH → pro → authority). Tracks K-factor
                        (referred signups / total users) and STRETCH conversion
                        lift. Generates proposals when stages drop below benchmark.
                        [Created v3.1.3-FINAL]

  viral_observer        Organic sharing and referral mechanics. Tracks share
                        surfaces (web Share API, clipboard, blog CTAs, social).
                        Computes K-factor (target: 0.35). Records referral chain
                        quality and generates proposals when K-factor is low.
                        [Created v3.1.3-FINAL]

  achievement_observer  Milestone tracking. Monitors writing streaks,
                        exercise completions, and profile achievements.

  attribution_tracker   UTM parameter tracking and referral source attribution.
                        Maps which blog posts and share surfaces drive signups.

  blog_observer         Blog performance. Tracks which posts drive the most
                        analysis starts and signup conversions.

-- Key Signals --

  conversion_ready      Emitted by stretch_observer when STRETCH completes.
                        Carries: user_id, signal_strength (0-1), tier, priority,
                        recommended_cta (pro_upgrade or authority_upgrade).

  funnel_underperformance  Emitted by funnel_observer when a stage drops >20%
                        below benchmark. Carries stage name, actual rate,
                        benchmark rate, gap, priority (high/medium).

  low_k_factor          Emitted by viral_observer when K < 0.175 (50% of target).
                        Triggers share card visibility proposal.

-- Admin WebSocket --

  wss://api.quirrely.com/ws/metrics
  Broadcasts live metrics every 5 seconds including STRETCH completions,
  MRR, active subscriptions, observer health scores.
